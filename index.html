HTML

  • HTML stands for Hypertext Markup Language.
  • HTML is a type of markup language used in the creation of webpages and web applications.
  • HTML consists of elements or tags that make up both the structure and content of a webpage.
  • HTML tags are enclosed by angle brackets (i.e., < >).
  • HTML attributes provide supplementary information about an element.
  • HTML is structured into different sections, such as the head and body elements, which both play significant roles in the composition of a webpage.
  • The head element contains inforamtion about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS is linked within the head element of an HTML file using the "link" tag.
  • CSS is used to define formatting and styling rules for elements of a webpage.
  • Rules define colours, asthetics, visual layout, and organization of elements on a webpage.
  • There are three different ways to style with CSS: Inline CSS, Internal CSS, and External CSS.
    • Inline CSS styles directly in an HTML tag using a style attribute.
    • Internal CSS embeds a style sheet in the head section of an HTML file using the style element.
    • External CSS creates a separate CSS document with the styling rules, which is then linked to the HTML file using the link element in the head section of the HTML file.
  • CSS rule consists of a selector, which defines the element or attribute, and declarations, which specify the property and value to be applied.
  • CSS declarations are enclosed by curly brackets (i.e., { }).
  • The margin property specifies the amount of space wanted outside of an element.
  • The padding property specifies the amount of space wanted around the content inside an element.
  • CSS uses the * selector to apply rules to all elements displayed on a page.

Git

  • Git is a version control system.
  • Git allows developers to undo changes and return to a previous version.
  • Git allows developers to collaborate remotely without overwriting each other's work.
  • Some common examples of git commands include:
    • git config --global user.name "[your name]": set up a name of the person writing the code
    • git config --global user.email "[valid-email]": set up an email address
    • git status: checks what branch we are currently on
    • git branch: provides a list of all the branches in the repository
    • git checkout [branch-name]: switch to another branch
    • git checkout -b [branch-name]: creates a new branch and switches to it
    • git add -A: adds all modifications in the current branch to the staging area
    • git commit -m "[descriptive message]": commit content in the staging area with a message
    • git pull: sync local branch with the remote branch (i.e., merge the most up-to-date commits from remote branch to local)
    • git push: update local commits with the remote branch (i.e., transmit local commits to the remote repo branch)

JavaScript

  • JavaScript is a high-level programming language that provides logic and interactivity for webpages.
  • JavaScript is linked in either the head or the body element of an HTML file using the "script" tag.
  • JavaScript is case sensitive.
  • A variable is a named container that allows us to store data in our code.
  • JavaScript has a number of fundamental data types, such as number, string, boolean, undefined, null, and symbol, as well as a single non-primitive data type known as object.
    • Number represents numeric data.
    • String represents sequence of text values enclosed in single ('') or double quotes ("").
    • Boolean represents logical values that can be either true or false.
    • Undefined represents a variable in absence of a value.
    • Null represents the absence of an object.
    • Symbol, also known as atoms, represents an unique identifier.
    • Object represents a collection of data, functions, properties and more.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript executes code from top to bottom unless interrupted by conditional statements, loops, or function calls.
  • In conditional statements, we check to see if the condition is falsy (i.e., 0, -0, 0n, "", null, undefined, NaN, or false) or truthy (i.e.,true; all other values that were not listed for falsy).
  • The Strict Equality Operator (===) evaluates whether two values are equal and returns a true or false result.
  • An array is a variable that represents a collection of related elements or data.
  • An array can store different types of data in a single variable.
  • The elements within an array can be accessed using their array index, also known as the location number or location identifier.
  • Arrays are zero-indexed and sequential, which means that the first item in any array starts with 0, followed by a predictable pattern of 1, 2, 3, and so on.
  • The "for" loop is commonly used with the predictable pattern of indices to perform a repeated task for each element in the array.
  • The "for" loop consists of 3 statements: the starting point of the loop, the condition that must be true for the loop to continue, and the statement that updates the loop variable on each iteration.
  • Commenting Out Code (/* code block */) inactivates code without deleting it.
  • Function is a set of instructions of a task.
  • JavaScript functions involve two main steps: defining the function with instructions, and then calling it to execute those instructions.
  • JavaScript function declaration (or the function definition) consists of a name followed by parentheses (), and a code block enclosed in curly brackets {}.
  • The function will not execute its code until we call it.
